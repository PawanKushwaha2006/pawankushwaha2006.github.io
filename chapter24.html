Chapter 24: Semantic HTML

Semantic tags describe their meaning to browsers, screen readers, and search engines. Replace generic <div> with meaningful tags. Better for SEO, accessibility, and code readability.

Old Way vs Modern Way:
Old: <div id="header"> - No meaning
Modern: <header> - Browser knows it's the page header

1. <header> - Page or Section Header

Represents introductory content. Can be used for whole page OR inside <article>, <section>.

Use Cases:

LocationContainsExample
Page HeaderLogo, site nav, searchTop of website
Article HeaderTitle, author, dateBlog post heading
Section HeaderSection titleInside <section>
<!-- Page Header -->
<header>
  <img src="logo.png" alt="Site Logo">
  <nav>
    <a href="/">Home</a>
    <a href="/blog">Blog</a>
  </nav>
</header>

<!-- Article Header -->
<article>
  <header>
    <h1>How to Learn HTML</h1>
    <p>By John Doe | May 1, 2026</p>
  </header>
  <p>Article content...</p>
</article>
Rules: You can have multiple <header> tags. Cannot nest <header> inside <header> or <footer>.

2. <footer> - Page or Section Footer

Contains info about its section: copyright, links, author info, back-to-top.

<!-- Page Footer -->
<footer>
  <p>© 2026 LearnWebDev. All rights reserved.</p>
  <nav>
    <a href="/privacy">Privacy</a>
    <a href="/terms">Terms</a>
    <a href="/contact">Contact</a>
  </nav>
</footer>

<!-- Article Footer -->
<article>
  <p>Article content...</p>
  <footer>
    <p>Tags: HTML, CSS, Web</p>
    <p>Author: Jane Smith</p>
  </footer>
</article>
Rules: Multiple <footer> tags allowed. Cannot nest inside <header> or another <footer>.

3. <article> - Self-Contained Content

Represents content that makes sense on its own. Should be distributable/reusable independently.

Use For:

Good for articleNOT for article
Blog postSite header
News articleSidebar
Forum postNavigation menu
Product cardCopyright footer
User commentBanner ad
<article>
  <header>
    <h2>HTML Semantic Tags Explained</h2>
    <p>Published: May 1, 2026 by Admin</p>
  </header>
  
  <p>Semantic HTML helps search engines understand...</p>
  
  <section>
    <h3>Why Use Semantic HTML?</h3>
    <p>Better SEO, accessibility...</p>
  </section>
  
  <footer>
    <p>Category: HTML | Comments: 24</p>
  </footer>
</article>
SEO Tip: Google treats each <article> as a separate entity. Use one <article> per blog post. Helps with rich snippets and featured snippets.

4. <aside> - Sidebar/Tangential Content

Content indirectly related to main content. Like sidebars, pull quotes, ads, related links, author bio.

<main>
  <article>
    <h1>Main Article</h1>
    <p>Main content here...</p>
  </article>
  
  <aside>
    <h3>Related Articles</h3>
    <ul>
      <li><a href="#">CSS Basics</a></li>
      <li><a href="#">JavaScript Intro</a></li>
    </ul>
    
    <h3>Advertisement</h3>
    <img src="ad.jpg" alt="Ad">
  </aside>
</main>
Use aside forDon't use aside for
Sidebar with related linksMain navigation
Author bio boxArticle main content
Pull quotesPage footer
AdvertisementsPrimary content
Table of contentsCopyright info

5. Complete Semantic Layout

This is the modern HTML5 page structure. Replace all divs with semantic tags.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Blog Page</title>
</head>
<body>

  <!-- Site Header -->
  <header>
    <img src="logo.png" alt="Site Logo">
    <nav>
      <a href="/">Home</a>
      <a href="/blog">Blog</a>
      <a href="/about">About</a>
    </nav>
  </header>

  <!-- Main Content Area -->
  <main>
    <!-- Blog Post -->
    <article>
      <header>
        <h1>Why Semantic HTML Matters</h1>
        <p>By John | May 1, 2026</p>
      </header>
      
      <section>
        <h2>Introduction</h2>
        <p>Semantic HTML improves...</p>
      </section>
      
      <section>
        <h2>SEO Benefits</h2>
        <p>Google understands structure...</p>
      </section>
      
      <footer>
        <p>Tags: HTML, SEO, Accessibility</p>
      </footer>
    </article>
    
    <!-- Sidebar -->
    <aside>
      <h3>Popular Posts</h3>
      <ul>
        <li><a href="#">CSS Grid Guide</a></li>
        <li><a href="#">JS Basics</a></li>
      </ul>
    </aside>
  </main>

  <!-- Site Footer -->
  <footer>
    <p>© 2026 LearnWebDev</p>
    <nav>
      <a href="/privacy">Privacy</a>
      <a href="/terms">Terms</a>
    </nav>
  </footer>

</body>
</html>
Visual Layout Preview:

<nav> Main Menu
<article>
<header> Article Title
<section> Content Block 1
<section> Content Block 2
<footer> Article Tags
<aside> Sidebar: Related Links, Ads
<footer> Copyright + Links

BenefitHow Semantic HTML Helps
SEOGoogle understands page structure. <article> = main content. <nav> = site links
AccessibilityScreen readers jump to <main>, <nav>, skip <aside>
Maintainability<header> clearer than <div class="header">

9. Div vs Semantic - When to Use What?

<!-- BAD: Too many divs -->
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">
  <div class="article">...</div>
  <div class="sidebar">...</div>
</div>
<div class="footer">...</div>

<!-- GOOD: Semantic -->
<header>...</header>
<nav>...</nav>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>
Rule of Thumb: If a tag describes the content's purpose, use semantic tag. Use <div> only for styling/layout wrappers with no meaning.

10. Summary - Golden Rules

  1. <header>: Intro content for page or section. Can have multiple. Logo, nav, title go here
  2. <footer>: Closing info for page or section. Copyright, links, author. Can have multiple
  3. <article>: Self-contained, reusable content. Blog post, product card, comment. Google loves this
  4. <aside>: Tangentially related content. Sidebars, ads, related links. Not main content
  5. <main>: Only one per page. Wraps primary content. Skip nav/header/footer
  6. Nesting: <article> can have <header> and <footer> inside
  7. SEO: Use semantic tags. Helps Google understand structure = better rankings
  8. Accessibility: Screen readers use landmarks to navigate. <nav> <main> <aside>
Don't Overuse: Not everything needs <section> or <article>. If it's just for styling, use <div>. If it has semantic meaning, use semantic tag.
