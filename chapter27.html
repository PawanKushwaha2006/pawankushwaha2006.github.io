Chapter 27: HTML File Paths

File paths tell the browser where to find files like images, CSS, JS, or other HTML pages. Wrong path = 404 error. There are 3 types: Absolute, Relative, and Root-relative.

1. Absolute Paths - Full URL

Complete web address starting with https://. Works from anywhere. Used for external resources.

Syntax:

<!-- External Image -->
<img src="https://example.com/images/logo.png" alt="Logo">

<!-- External CSS -->
<link rel="stylesheet" href="https://cdn.example.com/style.css">

<!-- External Link -->
<a href="https://google.com">Google</a>
ProsCons
Works from any pageBreaks if domain changes
Good for CDN, external filesLonger to type
Always points to same locationCan't test offline
Don't use for internal files: https://yoursite.com/style.css breaks on localhost. Use relative paths for your own files.

2. Relative Paths - Based on Current File

Path relative to the current HTML file's location. Most common for internal files.

Example Folder Structure:

project/ ├── index.html ← You are here ├── about.html ├── css/ │ └── style.css ├── js/ │ └── script.js └── images/ ├── logo.png └── icons/ └── home.svg

Case 1: Same Folder

File is in same folder as current HTML file.

<!-- From index.html to about.html -->
<a href="about.html">About Page</a>

<!-- Or with ./ -->
<a href="./about.html">About Page</a>

Case 2: Subfolder - Go Down

File is inside a folder. Use folder/file

<!-- From index.html to css/style.css -->
<link rel="stylesheet" href="css/style.css">

<!-- From index.html to images/logo.png -->
<img src="images/logo.png" alt="Logo">

<!-- From index.html to images/icons/home.svg -->
<img src="images/icons/home.svg" alt="Home">

Case 3: Parent Folder - Go Up

File is in parent folder. Use ../ to go up one level.

<!-- From css/style.css to index.html -->
<a href="../index.html">Home</a>

<!-- From js/script.js to images/logo.png -->
<img src="../images/logo.png" alt="Logo">

<!-- Go up 2 levels -->
<a href="../../index.html">Home</a>
Visual Examples:

Rule: ./ = current folder, ../ = go up one folder. You can chain: ../../images/pic.jpg

3. Root-Relative Paths - Start from Domain Root

Starts with /. Always starts from website root domain. Works regardless of current folder depth.

<!-- Always points to: https://yoursite.com/css/style.css -->
<link rel="stylesheet" href="/css/style.css">

<!-- Always points to: https://yoursite.com/images/logo.png -->
<img src="/images/logo.png" alt="Logo">

<!-- Always points to: https://yoursite.com/about.html -->
<a href="/about.html">About</a>
ProsCons
Works from any folder depthDoesn't work on localhost file://
No need to count ../Needs server to work
Easy to move files aroundBreaks if site in subfolder
Localhost Issue: Root-relative paths fail when opening HTML directly as file. /css/style.css looks for file:///css/style.css which doesn't exist. Use live server or relative paths for local dev.

5. Real Project Example

Folder structure for a blog website:

blog/ ├── index.html ├── about.html ├── assets/ │ ├── css/ │ │ └── main.css │ ├── js/ │ │ └── app.js │ └── images/ │ └── hero.jpg └── posts/ └── first-post.html ← You are here

From posts/first-post.html:

<!-- Link to homepage - Go up 1 level -->
<a href="../index.html">Home</a>

<!-- Link to CSS - Go up 1, then into assets/css -->
<link rel="stylesheet" href="../assets/css/main.css">

<!-- Link to JS -->
<script src="../assets/js/app.js"></script>

<!-- Image -->
<img src="../assets/images/hero.jpg" alt="Hero">

<!-- OR using root-relative - Works on server -->
<link rel="stylesheet" href="/assets/css/main.css">
<img src="/assets/images/hero.jpg" alt="Hero">

8. Best Practices

  1. Use relative for internal: css/style.css works everywhere - local + live
  2. Use absolute for external: https://cdn.com/jquery.js for CDN
  3. Lowercase filenames: logo.png not Logo.PNG - prevents case issues
  4. No spaces: Use hyphens: hero-image.jpg not hero image.jpg
  5. Organize folders: /assets/css, /assets/js, /assets/images
  6. Test paths: Open DevTools Console. 404 errors show broken paths
  7. Root-relative on server: Use /css/style.css only on live sites
  8. Consistent structure: Keep similar depth to avoid ../../../

9. Debugging Broken Paths

404 Error Checklist:
  1. Right-click page → Inspect → Console tab. Look for 404 errors
  2. Check spelling: stlye.css vs style.css
  3. Check case: Logo.png vs logo.png
  4. Check location: Is file really in that folder?
  5. Check slashes: Use / not \
  6. Try absolute path temporarily to test if file exists

10. Summary - Golden Rules

  1. Relative: folder/file.css - Most common, works locally and live
  2. Go up: ../ for each level up. ../../ = up 2 levels
  3. Absolute: https://... - Only for external CDN files
  4. Root-relative: /folder/file.css - Only on server, not local files
  5. Same folder: Just file.html or ./file.html
  6. No spaces/caps: Use lowercase-hyphens: my-file.jpg
  7. Forward slash: Always / never \ on web
  8. Test: Check Console for 404 errors after deploying
Pro Tip: When in doubt, right-click broken image → Open image in new tab. URL bar shows what path browser tried. Compare with actual file location.
