FUNCTION - COMPLETE GUIDE

A function is a rule that converts input to output. It's used in Math and Programming. The concept is the same everywhere - Give Input, Process it, Get Output.

📊 What is a Function in Math?
A function is a special relationship where every input has exactly one output. Like a vending machine - insert 10 rupees and you get exactly one chips packet, not two.
f(x) = y
x = Input (Domain), y = Output (Range), f = Function Rule
🎯 Real Life Examples
Example 1: Double Machine
f(x) = 2x
Input 5 → Output 10
Input 7 → Output 14
Example 2: Square Function
f(x) = x²
Input 3 → Output 9
Input 4 → Output 16
Example 3: Temperature
f(C) = (C × 9/5) + 32
Celsius to Fahrenheit
0°C → 32°F
📋 Types of Functions
Linear Function
f(x) = mx + c
Straight line graph
Ex: f(x) = 2x + 3
Quadratic Function
f(x) = ax² + bx + c
Parabola shape
Ex: f(x) = x² - 4
Exponential
f(x) = aˣ
Grows very fast
Ex: f(x) = 2ˣ
Trigonometric
sin(x), cos(x), tan(x)
Wave patterns
Used in physics
💡 Important Rule
Vertical Line Test: Draw a vertical line on the graph. If it cuts at more than one point, it's not a function. Every x must have only one y.
💻 What is a Function in Programming?
In programming, a function is a reusable block of code that performs a specific task. Write once, use multiple times. Code stays clean, no repetition needed.
function name(parameters) { code }
Define it → Call it → Get Result
🎯 JavaScript Examples
Example 1: Simple Function
function greet(name) { return "Hello " + name; } console.log(greet("Rahul")); // Output: Hello Rahul
Example 2: Math Function
function add(a, b) { return a + b; } let result = add(5, 3); // result = 8
Example 3: Arrow Function (Modern)
const square = (x) => x * x; console.log(square(4)); // Output: 16
🔧 Parts of a Function
1. Function Name
Unique name - greet, add, calculate. Use camelCase.
2. Parameters
Input values - (a, b). Used inside the function.
3. Function Body
Code inside { }. This is where the actual work happens.
4. Return Value
Output that function gives back. Use return keyword.
🎁 Benefits of Functions
1. Reusability - Write once, use 100 times
2. Organized Code - Code stays clean, easy to understand
3. Easy Debugging - Bug fixing is easy, change in one place
4. Less Code - No need to repeat
5. Modularity - Break large programs into smaller parts
💡 Pro Tip
Always start function names with a verb: getUserData, calculateTotal, validateEmail. This makes it clear what the function does. Don't use nouns like data, total.
