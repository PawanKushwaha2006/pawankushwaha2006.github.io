Chapter 18: Form Validation

HTML5 validation attributes let the browser check user input before submitting. No JavaScript needed for basic checks. Stops bad data at the client level.

Important: Client-side validation is for UX only. Always validate on server too. Users can bypass HTML validation.

1. The required Attribute

Makes a field mandatory. Form won't submit if empty. Works on input, select, textarea.

Basic Syntax:

<label for="email">Email:</label>
<input type="email" id="email" name="email" required>

2. min and max Attributes

Set minimum and maximum values. Works on number, range, date, time, month, week.

For Numbers:

<label for="age">Age (18-100):</label>
<input type="number" id="age" name="age" min="18" max="100" required>

3. minlength and maxlength

Control text length. Works on text, email, password, textarea, tel, url, search.

<label for="username">Username (3-15 chars):</label>
<input type="text" id="username" name="username" 
       minlength="3" maxlength="15" required>

maxlength prevents typing more characters. minlength shows error on submit if too short.

5. step Attribute

Defines legal number intervals. Works with number, range, date, time.

<!-- Only multiples of 5 -->
<input type="number" min="0" max="100" step="5">

<!-- Decimals allowed -->
<input type="number" step="0.01">

<!-- Weekly dates only -->
<input type="date" step="7">

7. CSS Pseudo-Classes for Validation

Style inputs based on validation state using CSS.

/* Valid input */
input:valid {
  border-color: #34d399;
}

/* Invalid input after user interacted */
input:invalid:not(:placeholder-shown) {
  border-color: #f87171;
}

/* Required fields */
input:required {
  background-image: url('star.svg');
}

/* Optional fields */
input:optional {
  opacity: 0.8;
}

9. The novalidate Attribute

Add to <form> to disable browser validation. Useful if you want custom JS validation only.

<form action="/submit" novalidate>
  <input type="email" required>
  <button>Submit</button>
</form>
Warning: Only use novalidate if you're handling validation with JavaScript. Otherwise users can submit invalid data.

10. Validation API with JavaScript

For custom messages and logic, use the Constraint Validation API.

<input type="text" id="custom" required>
<span id="error"></span>

<script>
const input = document.getElementById('custom');
const error = document.getElementById('error');

input.addEventListener('input', () => {
  if (input.validity.valueMissing) {
    error.textContent = 'This field is required';
  } else if (input.validity.tooShort) {
    error.textContent = 'Too short';
  } else {
    error.textContent = '';
  }
});
</script>

Validity Properties:

PropertyTrue When
valueMissingRequired field is empty
typeMismatchWrong type like invalid email
patternMismatchDoesn't match pattern
tooLongExceeds maxlength
tooShortBelow minlength
rangeUnderflowBelow min
rangeOverflowAbove max
stepMismatchDoesn't match step interval
validAll checks pass

12. Summary - Golden Rules

  1. required: Use on all mandatory fields. Form won't submit if empty
  2. min/max: For numbers and dates. Prevents out-of-range values
  3. minlength/maxlength: For text length. maxlength blocks typing, minlength checks on submit
  4. pattern: Use regex for custom formats like phone, zip, username
  5. step: Control increments for number, range, date inputs
  6. CSS :valid/:invalid: Style inputs based on validation state for better UX
  7. Help text: Always explain format requirements to users
  8. Server validation: HTML validation is for UX only. Backend must validate too
Pro Tip: Combine attributes: <input type="email" required minlength="5" maxlength="50"> for robust validation.
