Chapter 14: HTML Forms (Basics)

What is an HTML Form?

HTML forms are used to collect data from users. The data can be sent to a server for processing. Forms are used for login, registration, contact, search, surveys, and more.

1. The <form> Element

The <form> tag is a container for all input elements. It has two important attributes:

  • action - Specifies where to send the form data
  • method - GET or POST method to send data
<form action="/submit" method="post">
  <!-- Form elements go here -->
</form>
GET vs POST:
GET - Data visible in URL, used for search
POST - Data hidden, used for passwords, sensitive info

2. The <input> Element

The most important form element. The type attribute decides what kind of input field appears.

Text Input - type="text"

Single-line text field for names, usernames, etc.

<label for="username">Username:</label>
<input type="text" id="username" name="username" placeholder="Enter username">

Password Input - type="password"

Characters are masked with dots or asterisks.

<label for="pwd">Password:</label>
<input type="password" id="pwd" name="password">

Email Input - type="email"

Validates email format automatically on mobile shows @ key.

<label for="email">Email:</label>
<input type="email" id="email" name="email" required>

Number Input - type="number"

Only accepts numbers. You can set min and max values.

<label for="age">Age:</label>
<input type="number" id="age" name="age" min="1" max="120">

3. The <label> Element

Labels improve accessibility. Clicking the label focuses the input. The for attribute must match input's id.

<label for="user_id">Username:</label>
<input type="text" id="user_id" name="username">

4. Radio Buttons - type="radio"

Let user select ONLY ONE option. All radios must have the same name attribute.

<p>Select your gender:</p>

<input type="radio" id="male" name="gender" value="male">
<label for="male" class="inline">Male</label>

<input type="radio" id="female" name="gender" value="female">
<label for="female" class="inline">Female</label>

<input type="radio" id="other" name="gender" value="other">
<label for="other" class="inline">Other</label>

5. Checkboxes - type="checkbox"

Let user select MULTIPLE options.

<p>Select your skills:</p>

<input type="checkbox" id="html" name="skills" value="html">
<label for="html" class="inline">HTML</label>

<input type="checkbox" id="css" name="skills" value="css">
<label for="css" class="inline">CSS</label>

<input type="checkbox" id="js" name="skills" value="javascript">
<label for="js" class="inline">JavaScript</label>

Select your skills:

6. Textarea - Multi-line Text

For comments, messages, descriptions. Use rows and cols to set size.

<label for="message">Your Message:</label>
<textarea id="message" name="message" rows="4" cols="50" placeholder="Type here..."></textarea>

7. Dropdown - <select> Element

Creates a dropdown list. <option> defines each choice.

<label for="country">Select Country:</label>
<select id="country" name="country">
  <option value="">-- Select --</option>
  <option value="usa">United States</option>
  <option value="uk">United Kingdom</option>
  <option value="india">India</option>
  <option value="canada">Canada</option>
</select>

8. Submit Button

Sends all form data to the server.

<input type="submit" value="Submit Form">
<!-- OR -->
<button type="submit">Submit</button>

9. Complete Form Example

<form action="/register" method="post">
  <label for="fname">Full Name:</label>
  <input type="text" id="fname" name="fullname" required>

  <label for="femail">Email:</label>
  <input type="email" id="femail" name="email" required>

  <label for="fpass">Password:</label>
  <input type="password" id="fpass" name="password" required>

  <p>Gender:</p>
  <input type="radio" id="m" name="gender" value="male">
  <label for="m" class="inline">Male</label>
  <input type="radio" id="f" name="gender" value="female">
  <label for="f" class="inline">Female</label>

  <label for="fcountry">Country:</label>
  <select id="fcountry" name="country">
    <option value="india">India</option>
    <option value="usa">USA</option>
  </select>

  <label for="bio">Bio:</label>
  <textarea id="bio" name="bio" rows="3"></textarea>

  <input type="checkbox" id="terms" name="terms" required>
  <label for="terms" class="inline">I agree to terms</label>

  <button type="submit">Register</button>
</form>

10. Important Input Attributes

  • name - Required. Data key sent to server
  • id - Links to label, used by JavaScript/CSS
  • value - Default value of input
  • placeholder - Hint text shown when empty
  • required - Field must be filled before submit
  • readonly - User cannot edit, but value is sent
  • disabled - Field is disabled, value not sent
  • maxlength - Max characters allowed
  • min / max - For number, date, range inputs
✅ Best Practices:
1. Always use <label> with for for accessibility
2. Every input must have a name attribute or it won't be sent
3. Use required for mandatory fields
4. Use placeholder for hints, not as replacement for labels
5. Use type="email" for emails - gives mobile keyboard with @
6. Group related fields with <fieldset> and <legend>
7. Always use POST for passwords and sensitive data

11. Common Mistakes ❌

<!-- Wrong: No name attribute -->
<input type="text" id="name"> <!-- Won't be submitted -->

<!-- Wrong: Label not connected -->
<label>Email</label>
<input type="email"> <!-- Missing for and id -->

<!-- Wrong: Radio buttons with different names -->
<input type="radio" name="gender1">Male
<input type="radio" name="gender2">Female <!-- Both can be selected -->

<!-- Correct: Same name -->
<input type="radio" name="gender">Male
<input type="radio" name="gender">Female
