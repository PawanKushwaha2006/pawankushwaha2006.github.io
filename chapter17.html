Chapter 17: Buttons & Actions

Introduction to HTML Buttons

Buttons are used to trigger actions in forms and web pages. There are 3 main types: Submit, Reset, and Custom buttons.

1. Submit Button

The submit button sends form data to the server. It triggers the form's action attribute.

Method 1: Using <input type="submit">

<form action="/submit" method="post">
  <label for="email">Email:</label>
  <input type="email" id="email" name="email">

  <input type="submit" value="Submit Form">
</form>

Method 2: Using <button type="submit">

<form action="/submit" method="post">
  <label for="user">Username:</label>
  <input type="text" id="user" name="username">

  <button type="submit">Submit Form</button>
</form>
💡 Key Difference:
<input type="submit"> - Self-closing, text via value attribute
<button> - Can contain HTML, icons, images inside

2. Reset Button

The reset button clears all form fields and returns them to default values.

<form>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name">

  <label for="mail">Email:</label>
  <input type="email" id="mail" name="email">

  <input type="submit" value="Submit">
  <input type="reset" value="Clear Form">
</form>

3. Custom Button with <button> Tag

The <button> tag is more flexible. You can add icons, images, and custom JavaScript actions.

Button Types:

  • type="submit" - Submits the form (default)
  • type="reset" - Resets the form
  • type="button" - Does nothing, used with JavaScript
<!-- Submit button -->
<button type="submit">Save Data</button>

<!-- Reset button -->
<button type="reset">Reset All</button>

<!-- Custom button with JavaScript -->
<button type="button" onclick="alert('Hello!')">Click Me</button>

<!-- Button with icon -->
<button type="button">
  <span>🔍</span> Search
</button>

4. Button Styling Examples

<!-- Danger Button -->
<button type="button" class="btn-danger">Delete Account</button>

<!-- Success Button -->
<button type="button" class="btn-success">Confirm</button>

<!-- Warning Button -->
<button type="button" class="btn-warning">Warning</button>

<!-- Disabled Button -->
<button type="button" disabled>Disabled</button>

5. Input vs Button - When to Use Which?

Use <input> when:
1. Simple text button needed
2. Old browser compatibility required
3. Form submission only

Use <button> when:
1. Need icons or HTML inside button
2. Custom styling required
3. JavaScript actions needed
4. Modern web apps

6. Complete Form with All Button Types

<form action="/register" method="post">
  <label for="fname">Full Name:</label>
  <input type="text" id="fname" name="fullname" required>

  <label for="femail">Email:</label>
  <input type="email" id="femail" name="email" required>

  <!-- Submit button -->
  <button type="submit">Register Now</button>

  <!-- Reset button -->
  <button type="reset">Clear Form</button>

  <!-- Custom button -->
  <button type="button" onclick="alert('Help: Fill all fields')">Help</button>
</form>

7. Button Attributes

  • type - submit, reset, or button
  • name - Name sent to server
  • value - Value sent to server
  • disabled - Makes button unclickable
  • form - Associates button with a form by id
  • onclick - JavaScript to run on click
✅ Best Practices:
1. Always specify type attribute for buttons
2. Use type="button" for custom JavaScript actions
3. Never use reset button without user confirmation
4. Make button text clear - "Submit" not "Click Here"
5. Disable submit button after click to prevent double submission
6. Use <button> for modern apps, more flexible than <input>

8. Common Mistakes ❌

<!-- Wrong: Button without type defaults to submit -->
<button>Click</button> <!-- Inside form, this will submit! -->

<!-- Correct: Specify type -->
<button type="button">Click</button>

<!-- Wrong: Using reset without warning -->
<input type="reset"> <!-- Users may lose data accidentally -->
