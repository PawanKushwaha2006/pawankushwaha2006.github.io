Chapter 20: HTML Audio

The <audio> element embeds sound content in documents. Used for music, podcasts, sound effects, voice notes. HTML5 native - no Flash needed.

1. Basic Syntax

<audio src="song.mp3" controls>
  Your browser does not support the audio tag.
</audio>

Key Points:
1. src = Path to audio file
2. controls = Shows play/pause/volume UI. Without it, audio is invisible
3. Text inside <audio> = Fallback for old browsers

2. Audio Attributes

AttributePurposeExample
srcURL of audio filesrc="music.mp3"
controlsShow browser controls<audio controls>
autoplayPlay automatically on load<audio autoplay>
loopRepeat audio when finished<audio loop>
mutedStart muted<audio muted>
preloadHow to load: auto/metadata/nonepreload="metadata"
controlslistHide specific controlscontrolslist="nodownload"

Preload Attribute Values:

ValueBehavior
noneDon't preload. Saves bandwidth
metadataLoad duration, dimensions only. Recommended
autoLoad entire file. Uses more data
Autoplay Policy: Most browsers block autoplay with sound. It only works if muted is also present, or after user interaction. This prevents annoying auto-playing ads.

3. Multiple Sources - <source>

Different browsers support different formats. Provide multiple sources for maximum compatibility. Browser picks first supported format.

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  <source src="audio.wav" type="audio/wav">
  Your browser does not support audio.
</audio>

Supported Audio Formats:

FormatMIME TypeBrowser SupportFile Size
MP3audio/mpegAll browsersSmall - Best choice
OGGaudio/oggChrome, Firefox, OperaSmall
WAVaudio/wavAll browsersLarge - Uncompressed
AACaudio/aacSafari, ChromeSmall
FLACaudio/flacChrome, FirefoxMedium - Lossless
Best Practice: Always provide MP3 as first source. It works everywhere. Add OGG as fallback for Firefox/Opera. Avoid WAV unless you need lossless quality.

4. Controlling Audio with JavaScript

The <audio> element has a JavaScript API for custom controls.

Key Properties & Methods:

const audio = document.querySelector('audio');

// Properties
audio.play();           // Start playing
audio.pause();          // Pause
audio.currentTime = 0;  // Seek to start
audio.volume = 0.5;     // 0.0 to 1.0
audio.muted = true;     // Mute/unmute
audio.playbackRate = 2; // 2x speed

// Properties - Read only
console.log(audio.duration);    // Total seconds
console.log(audio.currentTime); // Current position
console.log(audio.paused);      // true/false
console.log(audio.ended);       // true/false

// Events
audio.addEventListener('play', () => console.log('Started'));
audio.addEventListener('pause', () => console.log('Paused'));
audio.addEventListener('ended', () => console.log('Finished'));
audio.addEventListener('timeupdate', () => console.log(audio.currentTime));

5. Custom Audio Player UI

Hide native controls and build your own for branding.

<audio id="player" src="song.mp3"></audio>

<div class="custom-player">
  <button id="playBtn">Play</button>
  <div class="progress-bar">
    <div class="progress-fill"></div>
  </div>
  <span class="time">0:00 / 0:00</span>
</div>

<script>
const audio = document.getElementById('player');
const playBtn = document.getElementById('playBtn');

playBtn.onclick = () => {
  if(audio.paused) {
    audio.play();
    playBtn.textContent = 'Pause';
  } else {
    audio.pause();
    playBtn.textContent = 'Play';
  }
};
</script>

6. Common Use Cases

Case 1: Background Music

<audio src="ambient.mp3" autoplay loop muted controls controlslist="nodownload">
</audio>

Autoplay muted, loops forever, user can unmute. controlslist="nodownload" hides download button.

Case 2: Podcast Player

<audio src="episode1.mp3" controls preload="metadata">
  <source src="episode1.mp3" type="audio/mpeg">
  <source src="episode1.ogg" type="audio/ogg">
</audio>

Preload metadata only to save bandwidth. Show controls for seek/speed.

Case 3: Sound Effects on Click

<audio id="clickSound" src="click.mp3" preload="auto"></audio>
<button onclick="document.getElementById('clickSound').play()">
  Click Me
</button>

7. Accessibility Best Practices

PracticeWhy
Always add controlsUsers need play/pause/volume. Keyboard users need focus
Provide captionsUse <track> for transcripts if speech
No autoplay with soundUnexpected audio is disruptive for screen readers
Label audioAdd aria-label="Podcast episode 5" to audio tag
Keyboard supportEnsure custom players work with Tab, Space, Arrow keys

Adding Captions with track:

<audio controls>
  <source src="podcast.mp3" type="audio/mpeg">
  <track kind="captions" src="podcast.vtt" srclang="en" label="English">
</audio>

8. Common Mistakes

MistakeProblemFix
No controls attributeAudio invisible, can't playAdd controls or build custom UI
Autoplay with soundBlocked by browsersAdd muted or wait for user click
Only one formatWon't work on all browsersProvide MP3 + OGG sources
Large WAV filesSlow loading, wastes dataUse MP3 or AAC for web
preload="auto" on many filesWastes bandwidthUse preload="metadata" or "none"
No fallback textOld browsers show nothingAdd text inside <audio> tag

9. Performance Tips

  1. Compress audio: Use MP3 at 128kbps for speech, 192kbps for music
  2. Use preload="metadata": Loads duration only, not full file
  3. Lazy load: Set src with JS when user clicks play
  4. CDN: Host audio on CDN like Cloudflare for faster delivery
  5. Streaming: For long files, use HLS or DASH instead of MP3

10. Summary - Golden Rules

  1. Always add controls: Unless building custom player
  2. Provide MP3: Universal format. Add OGG for Firefox/Opera fallback
  3. Don't autoplay sound: Browsers block it. Use muted or wait for click
  4. Use preload="metadata": Best balance of UX and bandwidth
  5. Loop for background: Add loop for ambient music
  6. JavaScript API: Use play(), pause(), currentTime for custom controls
  7. Accessibility: Add labels, captions, keyboard support
  8. Mobile: Test on real devices. iOS requires user interaction to play
Pro Tip: For playlists, don't load multiple <audio> tags. Use one tag and change src with JavaScript when track ends.
