Chapter 25: Block vs Inline Elements

1. The Core Difference

Every HTML element has a default display behavior: Block or Inline. This decides how the element appears on the page.

Block Element Inline Element
Starts on a new line Stays in the same line
Takes full width available Takes only required width
Width & height can be set Width & height ignored
Margin/padding work on all sides Top/bottom margin don't work
Examples: <div>, <p>, <h1> Examples: <span>, <a>, <strong>

2. Block Elements - <div>

<div> = Division. It's a generic block-level container. Used to group large sections of content.

<div>This is div 1</div>
<div>This is div 2</div>
<div>This is div 3</div>

Output:

This is div 1 - Takes full width
This is div 2 - Starts on new line
This is div 3 - Each div on separate line

3. Inline Elements - <span>

<span> is a generic inline container. Used to style small parts of text within a line.

<p>This is <span>span 1</span> <span>span 2</span> <span>span 3</span> in one line.</p>

Output:

This is span 1 span 2 span 3 in one line.

4. Visual Comparison

<!-- Block: Stacks vertically -->
<div style="background:blue;">Block 1</div>
<div style="background:red;">Block 2</div>

<!-- Inline: Flows horizontally -->
<span style="background:blue;">Inline 1</span>
<span style="background:red;">Inline 2</span>

Output:

Block 1
Block 2
Inline 1 Inline 2

5. Width & Height Behavior

<!-- Block: width/height works -->
<div style="width:200px; height:100px; background:blue;">Block</div>

<!-- Inline: width/height ignored -->
<span style="width:200px; height:100px; background:red;">Inline</span>
Block: 200x100 works
Inline: width/height ignored, only content size

6. Margin & Padding Behavior

<!-- Block: margin works on all sides -->
<div style="margin:20px; background:blue;">Block</div>

<!-- Inline: top/bottom margin doesn't work -->
<span style="margin:20px; background:red;">Inline</span>
Important: Inline elements ignore margin-top and margin-bottom. Only left/right margins work. But padding works on all sides.

7. Real-World Use Cases

When to use <div>:

<!-- Page layout sections -->
<div class="header">...</div>
<div class="sidebar">...</div>
<div class="content">...</div>
<div class="footer">...</div>

<!-- Card container -->
<div class="card">
  <h3>Title</h3>
  <p>Description</p>
</div>

When to use <span>:

<!-- Highlight part of text -->
<p>This is <span style="color:red;">important</span> text.</p>

<!-- Style individual words -->
<p>Price: <span class="price">$99</span></p>

<!-- Icons inside text -->
<p>Click <span>🔍</span> to search</p>

This is important text.

Price: $99

Click 🔍 to search

8. Nesting Rules

<!-- Correct: Block can contain inline and block -->
<div>
  <p>Paragraph</p>
  <span>Span inside div</span>
</div>

<!-- Correct: Inline can contain only inline -->
<span>
  <strong>Bold</strong>
  <a href="#">Link</a>
</span>

<!-- Wrong: Inline cannot contain block -->
<span>
  <div>This is invalid</div> <!-- Don't do this -->
</span>

9. Common Block Elements

  • <div>, <p>, <h1> to <h6>
  • <header>, <footer>, <section>, <article>
  • <ul>, <ol>, <li>
  • <form>, <table>, <hr>

10. Common Inline Elements

  • <span>, <a>, <strong>, <em>
  • <img>, <input>, <button>
  • <label>, <code>, <br>

11. Changing Display with CSS

You can change default behavior using CSS display property:

<!-- Make div inline -->
<div style="display:inline;">Now I'm inline</div>

<!-- Make span block -->
<span style="display:block;">Now I'm block</span>

<!-- Make anything inline-block -->
<div style="display:inline-block; width:100px;">Best of both</div>
✅ Best Practices:
1. Use <div> for layout and grouping sections
2. Use <span> for styling small text parts
3. Don't use <div> when semantic tag exists - use <header>, <nav>
4. Don't put block elements inside inline elements
5. Use display:inline-block when you need inline flow + width/height
6. Remember: Block = vertical stack, Inline = horizontal flow

12. Quick Summary

Div = Container for blocks. Starts new line, full width. Use for layout.

Span = Container for text. Stays inline, content width. Use for styling words.
