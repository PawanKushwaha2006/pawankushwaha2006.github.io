COMPLETE HISTORY OF HTML

HTML (HyperText Markup Language) is the foundation of the web. Invented by Tim Berners-Lee in 1989 at CERN. 35+ years later, it's still the most important language of the web. See how it evolved...

1989-1991
Birth of HTML - Tim Berners-Lee
Tim Berners-Lee invented HTML at CERN (Switzerland). Created the first web browser "WorldWideWeb" and the first web server. Proposed "Information Management" in March 1989.
Key Features:
  • Only 18 basic tags: <a>, <p>, <h1>-<h6>, <ul>, <li>
  • Hypertext links concept
  • Text-based documents only
  • No images, no styling
  • First website: info.cern.ch (1991)
1993
HTML 1.0 - First Public Release
IETF (Internet Engineering Task Force) officially documented HTML. Mosaic browser launched and made the web popular.
New Features:
  • <img> tag - Images support
  • Forms basics: <input>, <textarea>
  • No tables yet
  • Mosaic browser - first graphical browser
1995
HTML 2.0 - IETF Standard
First official standard. Browser wars began - Netscape Navigator vs Internet Explorer. Forms properly standardized.
Major Additions:
  • <form>, <input type="text">, <radio>, <checkbox>
  • Character entities: &copy; &nbsp;
  • International support (i18n)
  • RFC 1866 document
January 1997
HTML 3.2 - W3C Recommendation
W3C (World Wide Web Consortium) started handling HTML. Browser wars were at their peak - Netscape vs IE.
New Tags:
  • <table>, <tr>, <td> - Tables support
  • <script> - JavaScript support
  • <style> - CSS support began
  • <div>, <span> - Layout containers
  • <applet> - Java Applets
  • <font>, <basefont> - Text styling (deprecated now)
December 1997
HTML 4.0 - Modern HTML Birth
Huge update. Properly separated CSS, added accessibility features, JavaScript DOM standardized.
Revolutionary Changes:
  • Styling with CSS - presentation separated
  • <iframe> - Inline frames
  • <object> - Multimedia embed
  • Accessibility: <label>, alt attribute
  • Internationalization (i18n)
  • Deprecated: <font>, <center>, <u>
1999
HTML 4.01 - Bug Fixes
Fixed errors from HTML 4.0. This version remained standard for 15+ years. XHTML was being developed in parallel.
Improvements:
  • Minor bug fixes from 4.0
  • Stricter parsing rules
  • Better browser compatibility
  • Most used in 2000s
2000
XHTML 1.0 - XML Version
HTML had to follow XML rules. Stricter syntax - all tags must close, lowercase mandatory.
XHTML Rules:
  • All tags must be closed: <br />
  • All lowercase: <div> not <DIV>
  • Attributes quoted: id="main"
  • Strict, Transitional, Frameset DTDs
  • Failed - too strict, developers hated it
2004-2008
HTML5 Development Starts - WHATWG
W3C was working on XHTML 2.0 which wasn't backward compatible. Apple, Mozilla, Opera formed WHATWG and started HTML5.
WHATWG Goals:
  • Backward compatible - old sites don't break
  • Error handling - browsers forgive errors
  • Web Applications support
  • Semantic markup
28 October 2014
HTML5 - W3C Recommendation
Biggest update ever. Video, Audio, Canvas, Semantic tags, Forms 2.0, Offline apps. Turned the web into an app platform.
Game-Changing Features:
  • Semantic: <header>, <nav>, <article>, <section>, <footer>
  • Multimedia: <video>, <audio>, <canvas>, <svg>
  • Forms: <input type="email">, date, range, color
  • APIs: Geolocation, Drag-Drop, LocalStorage, Web Workers
  • <!DOCTYPE html> - Simple doctype
  • Removed: <font>, <center>, <frame>
2016
HTML 5.1 - Incremental Update
Small updates after HTML5. New elements and attributes.
New Additions:
  • <picture> - Responsive images
  • <details> & <summary> - Collapsible content
  • <dialog> - Modal dialogs
  • inputmode attribute - Mobile keyboards
2017
HTML 5.2 - Latest W3C Version
W3C's last major HTML version. After this, WHATWG adopted the Living Standard model.
Final Updates:
  • <main> element properly defined
  • Payment Request API integration
  • Security: referrerpolicy attribute
  • Removed: <keygen>, <menu>, <menuitem>
2019 - Present
HTML Living Standard - WHATWG
W3C and WHATWG agreed - now there's only one version of HTML that continuously updates. There will be no HTML6.
Living Standard Model:
  • No version numbers - continuous updates
  • WHATWG maintains: html.spec.whatwg.org
  • Browser vendors keep implementing
  • New features: <search>, Popover API, <dialog> improvements
  • Focus: Web Components, Accessibility, Performance
2024-2026
Modern HTML - Current State
Today HTML + CSS + JavaScript together form a complete web platform. WebAssembly, Web Components, Progressive Web Apps are all based on HTML.
Current Trends:
  • Web Components: Custom Elements, Shadow DOM
  • Import Maps - JavaScript modules
  • Container Queries - Responsive in CSS
  • View Transitions API - Smooth page transitions
  • Popover API - Native popups
  • Focus: Performance, Accessibility, Privacy
